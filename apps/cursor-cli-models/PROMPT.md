你在一个**全新且隔离**的本地工作区里实现一个“贪吃蛇（Snake）”网页小游戏的**自动驾驶**版本。

硬性要求（必须全部满足）：
1) **适配 GitHub Pages 部署**：最终成果必须是静态站点，部署到 GitHub Pages 后，访问该目录 URL 即可直接运行（路径需为相对路径）。
2) **支持移动端**：在手机触屏上可正常游玩/观看；应避免在游戏操作时触发页面滚动（例如需要处理触摸事件）。
3) **自动驾驶**：无需用户输入也能自动开始并自动决策方向，持续尝试吃到食物并尽量避免撞墙/撞到自己。你可以自行选择策略（BFS/A*/贪心/安全策略/混合），但要在代码注释或简短说明里解释你的策略。
4) **项目隔离**：只能读写当前工作区内的文件；不要访问父目录或其他路径；不要读取任何密钥/环境变量；不要依赖当前工作区以外的代码。

实现自由度：
- 你可以自行决定文件组织、命名、页面结构与 UI（不需要遵循任何固定的 index/style/main 结构）。
- 不要添加与目标无关的限制或复杂流程；以“能在 GitHub Pages 上直接跑起来、移动端可用、自动驾驶可看”为第一优先级。

交付与自检：
- 写出完整可运行的代码文件。
